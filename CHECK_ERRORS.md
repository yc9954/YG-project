# 에러 확인 가이드

## 🔍 안무 생성 실패 시 확인 방법

### 1. 백엔드 터미널 확인

백엔드 서버를 실행한 터미널에서 다음 로그를 확인하세요:

#### 정상 작동 시:
```
📥 모션 생성 요청 수신:
   프롬프트: ...
   스타일: ...
🎬 모션 생성 시작 (job_id: ...)
   오디오 분석 중...
   모션 생성 중...
✅ 모션 생성 완료 (job_id: ...)
```

#### 에러 발생 시:
```
❌ 모션 생성 오류 (job_id: ...):
   에러 메시지: [실제 에러]
   상세 트레이스:
   [스택 트레이스]
```

### 2. 프론트엔드 콘솔 확인

브라우저 개발자 도구(F12) → Console 탭:
- `🎬 모션 생성 시작`
- `✅ 모션 생성 완료`
- `❌ 모션 생성 오류`

### 3. 네트워크 탭 확인

브라우저 개발자 도구 → Network 탭:
- `/api/generate-motion` 요청 찾기
- Status 코드 확인 (200 = 성공, 500 = 서버 에러)
- Response 확인

---

## 🐛 일반적인 에러들

### 에러 1: "Audio analysis failed"
**원인**: 오디오 파일 분석 실패
**해결**: 오디오 파일 형식 확인 (mp3, wav 등)

### 에러 2: "Motion generation failed"
**원인**: 모션 생성 중 예외 발생
**해결**: 백엔드 로그에서 상세 에러 확인

### 에러 3: 타임아웃
**원인**: 생성 시간이 너무 오래 걸림
**해결**: 더 짧은 오디오 파일로 테스트

---

## ✅ 정상 작동 확인

다음이 모두 보이면 정상:
1. 백엔드: `✅ 모션 생성 완료`
2. 프론트엔드: `✅ 모션 생성 완료`
3. 타임라인에 모션이 추가됨

---

## 💡 디버깅 팁

1. **항상 백엔드 터미널 확인**: 가장 중요한 로그
2. **프론트엔드 콘솔 확인**: 클라이언트 측 에러
3. **네트워크 탭 확인**: API 요청/응답 확인
4. **에러 메시지 복사**: 정확한 에러 메시지를 알려주면 해결 가능
